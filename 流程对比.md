# 流程对比：当前项目 vs 参考项目

| 步骤 | 参考项目 | 当前项目 | 状态 |
|------|---------|---------|------|

| 2. Milvus 初始化 | `milvus_db_manager.initialize()` | 无 | - |
| 3. Redis 初始化 | `redis_manager.initialize()` | 无 | - |
| 4. Neo4j 初始化 | `neo4j_manager.initialize()` | 有 | OK |
| 5. MinIO 初始化 | `Minio(...)` → `app.state.minio_client` | 无（file_service 自己初始化） | 可接受 |
| 6. 模型网关 | `ModelGateway()` → 注册多模型（LangChain + LlamaIndex 双接口） | `ModelGateway()` → 注册单模型 | 部分 |
| 7. Embedding/Rerank | 本地加载 embed_model + rerank_model | 有 embedding_service | OK |
| 8. 检索管线 | `RetrievalPipeline(embed, rerank, gateway)` | 有 retrieval_pipeline | OK |
| 9. SubAgent 发现 | `discover_specialist_agents()` → `app.state.specialist_subagents` | `_build_subagents_from_specialists()` → `app.state.specialist_subagents` | OK |
| 10. Triage DeepAgent | `async_create_agent(..., checkpointer, store, subagents, gateway)` → `app.state.agent` | `create_triage_agent(..., subagents, gateway)` → `app.state.agent` | **缺 checkpointer/store** |
| 11. Executor DeepAgent | `async_create_agent(..., interrupt_on, checkpointer, store)` → `app.state.executor_agent` | `create_executor_agent(..., interrupt_on)` → `app.state.executor_agent` | **缺 checkpointer/store** |

| 13. TaskContextManager | `TaskContextManager(store)` → `app.state.context_manager` | 无 | 缺失 |
| 14. TaskExecutor | `TaskExecutor(store, event_bus, executor_agent, context_manager)` → `app.state.task_executor` | `task_executor` 全局变量，未注入 executor_agent | **未关联 executor_agent** |
| 15. DeadLetterQueue | `DeadLetterQueue(store)` → 后台扫描 | 全局变量，未启动扫描 | 部分 |
| 16. 工具热加载 | `ToolHotReloader(on_reload=_rebuild_agents)` | `ToolHotReloader(on_reload=_rebuild_agents)` | OK |
| 17. SubAgent 热加载 | `SubAgentHotReloader(on_reload=_rebuild_agents)` | 无 | 缺失 |
| 18. 恢复后台任务 | `task_executor.recover_tasks()` | 无 | 缺失 |

## 五、参考项目中 interinterrupt_on 的实际效果

参考项目 Executor 创建时传了 `interrupt_on={"request_approval": True}`。
当 LLM 调用 `request_approval` 工具时，`HumanInTheLoopMiddleware` 通过 LangGraph `interrupt()` 挂起执行。
当前项目传了但 deepagents 需要 `request_approval` 工具实际存在——目前 `TOOL_REGISTRY` 中没有这个工具。

## 六、缺失项汇总（按优先级）

### 必须修复
1. **对话历史**：checkpointer 未初始化 → 跨轮对话不工作
2. **会话保存**：/chat 路由不写 agentx_session → 会话列表始终为空
3. **TaskExecutor 关联 executor_agent**：后台任务每次 new agent，应复用 `app.state.executor_agent`
4. **request_approval 工具**：TOOL_REGISTRY 中不存在，interrupt_on 形同虚设

### 应该修复
5. **MinIO 初始化**：移到 lifespan，存 `app.state.minio_client`
6. **SubAgent 热加载**：新增 `SubAgentHotReloader`
7. **会话标题生成**：/chat 完成后 LLM 生成标题写入 agentx_session
8. **用户反馈端点**：/feedback 路由

### 可选优化
9. PG checkpointer/store（需要安装 pgvector + 配置 PG 连接池）
10. EventBus 实际使用（Redis Pub/Sub）
11. DeadLetter 扫描启动
