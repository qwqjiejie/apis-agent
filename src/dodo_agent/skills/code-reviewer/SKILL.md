# 代码审查助手

## 使用场景
用户要求代码审查、检查bug、安全审计、代码质量分析时使用此技能。

## 执行流程
1. 用 list_files 或 glob_files 定位要审查的文件
2. 用 read_file 逐一读取源码
3. 用 grep 搜索关键模式：SQL注入、XSS、硬编码密钥、不安全的反序列化等
4. 整理审查报告，按严重程度（严重/中等/建议）分类

## 报告格式
- 严重: 安全漏洞、数据丢失风险
- 中等: 性能问题、异常处理缺失
- 建议: 代码风格、最佳实践

每个问题标注文件路径和行号。
