# 重构计划

> 基于 Multi-Agent-UniversalFramework（主参考）+ dodo-agentx（辅助参考）的完整对齐重构方案

---

## 一、重构目标

将 apis-agent 从 "类继承 + 多入口 + LangChain ReAct" 架构重构为 "声明式 + 单入口 + deepagents 框架" 架构，与 Multi-Agent-UniversalFramework 架构对齐，同时吸收 dodo-agentx 的优势设计。

### 核心变更

| 维度 | 现状 | 目标 |
|------|------|------|
| Agent 框架 | LangChain `create_agent` (ReAct) | `deepagents.create_deep_agent` |
| Agent 入口 | 7 个独立端点 | 1 个 `/api/v1/chat` 统一入口 |
| Agent 扩展 | 类继承 BaseAgent + 路由注册 | AGENT.md 声明式（子代理目录） |
| 路由方式 | URL 硬编码 | 能力前缀 + LLM 自路由 |
| 数据库 | MySQL (SQLAlchemy + pymysql) | PostgreSQL (asyncpg + psycopg) |
| 中间件 | 无 | 完整中间件管线 |
| 子代理 | prompt 文本注入 | 原生 SubAgent spawning |
| 后台任务 | 简化版 TaskExecutor | 完整 TaskExecutor（HITL/快照/恢复/Journal/死信） |
| 技能管理 | 无 DB 管理 | Skill 生命周期 DB 管理 + zip 分发 |

## 二、能力前缀协议

### 2.1 设计说明

合并为单入口后，前端通过以下方式传递能力选择：在用户输入的最前面添加能力前缀标记，格式为 `能力名: 用户原始输入`。

### 2.2 能力前缀列表

| 前缀 | 对应能力 | 对应的 Specialist |
|------|---------|------------------|
| `生成ppt:` | PPT 自动生成 | ppt_specialist |
| `深度研究:` | 深度研究报告 | research_specialist |
| `分析文档:` | 文件/文档分析 | file_analysis_specialist |
| `数据分析:` | 数据库查询分析 | data_analysis_specialist |
| `代码审查:` | 代码审查 | code_review_specialist |
| `写代码:` | 编写代码/脚本 | coding_specialist |
| (无前缀) | 通用问答 | Triage 自行判断分流 |

### 2.3 前端发送示例

```json
// 生成 PPT
{ "message": "生成ppt: 帮我做一份关于AI行业2025年发展趋势的PPT", "conversationId": "xxx" }

// 深度研究
{ "message": "深度研究: 量子计算在金融领域的应用前景", "conversationId": "xxx" }

// 通用问答（无前缀）
{ "message": "今天天气怎么样", "conversationId": "xxx" }

// 文件分析
{ "message": "分析文档: 帮我总结这份合同的关键条款", "conversationId": "xxx", "fileIds": ["file-xxx"] }
```

### 2.4 后端处理

TriageAgent 收到消息后，首先检查能力前缀：
- 匹配到前缀 → 在 system prompt 中高优提示该 Specialist，引导 LLM 优先调用对应的 `task` 工具
- 未匹配 → LLM 自行判断分流（简单问题直接回答 / 复杂任务后台执行）

## 三、P0 — 基础架构改造（阻塞项，必须最先完成）

### P0-1: 数据库迁移 MySQL → PostgreSQL

**目标**：从 MySQL + SQLAlchemy 迁移到 PostgreSQL + asyncpg + psycopg，对齐 UniversalFramework 的 LangGraph checkpointer + Store 架构。

**迁移原因**：
1. LangGraph 的 `AsyncPostgresSaver` (checkpointer) 和 `AsyncPostgresStore` 只支持 PostgreSQL
2. deepagents 框架依赖 LangGraph checkpointer/store 进行状态持久化和跨会话记忆
3. UniversalFramework 基于 PostgreSQL 双库架构（auth_db + conversations_db），多租户/认证依赖 PostgreSQL

**新增依赖** (`pyproject.toml`):

```toml
"asyncpg>=0.30.0",
"psycopg>=3.2.0",
"psycopg-pool>=3.2.0",
"psycopg-binary>=3.2.0",
"langgraph-checkpoint-postgres>=2.0.0",
"langgraph-store-postgres>=0.2.0",
"deepagents>=0.1.0",
```

**移除依赖**:

```toml
# 移除
"pymysql>=1.1.0",
```

### P0-3: 引入 deepagents 框架，统一 Agent 工厂

**目标**：用 `deepagents.create_deep_agent` 替换 `langchain.agents.create_agent`，实现统一工厂 + 中间件管线。

**新建文件**：`src/apis_agent/agent/agent_factory.py`

```python
# agent_factory.py — Agent 统一工厂
# 基于 deepagents.create_deep_agent + 领域定制中间件

async def create_agent(
    model_name: str,
    fallback_model_name: str,
    tools: list,
    system_prompt: str,
    checkpointer,
    store,
    context_schema=None,
    subagents: list | None = None,
    gateway=None,
    interrupt_on: dict | None = None,
    extra_middleware: list | None = None,
) -> CompiledStateGraph:
    """统一的 Agent 工厂函数。
    
    Triage 和 Executor 使用同一个工厂，仅参数不同。
    
    Args:
        model_name: 主模型名
        fallback_model_name: 降级模型名
        tools: 工具列表
        system_prompt: 系统提示词
        checkpointer: LangGraph checkpointer（PostgreSQL）
        store: LangGraph store（PostgreSQL）
        context_schema: 上下文 Pydantic 模型
        subagents: Specialist Sub-Agent 列表
        gateway: 模型网关实例
        interrupt_on: HITL 中断配置（如 {"request_approval": True}）
        extra_middleware: 额外的定制中间件
    """
    from deepagents import create_deep_agent
    from deepagents.backends import StateBackend, StoreBackend, CompositeBackend
    from langchain.agents.middleware import (
        ModelCallLimitMiddleware,
        ToolCallLimitMiddleware,
        ModelFallbackMiddleware,
        ToolRetryMiddleware,
        ModelRetryMiddleware,
    )
    from src.apis_agent.gateway.middleware import GatewayMiddleware
    from src.apis_agent.gateway.types import ModelRole

    # 模型实例
    langchain_llm = await build_langchain_llm(model_name)
    fallback_llm = await build_langchain_llm(fallback_model_name) if gateway is None else None

    # CompositeBackend — 文件/技能/记忆统一后端路由
    backend = CompositeBackend(
        default=StateBackend(),
        routes={
            "/memories/": StoreBackend(
                store=store,
                namespace=lambda rt: (
                    "memories",
                    str(get_config()["configurable"].get("user_id", "anonymous")),
                ),
            ),
        },
    )

    # 领域定制中间件
    user_middleware = [
        ModelCallLimitMiddleware(
            run_limit=15, thread_limit=100, exit_behavior="end"
        ),
        # 智能网关路由
        GatewayMiddleware(gateway=gateway, role=ModelRole.CHAT)
        if gateway is not None
        else ModelFallbackMiddleware(first_model=fallback_llm),
        ModelRetryMiddleware(max_retries=1, backoff_factor=1.0, initial_delay=0.5),
        ToolCallLimitMiddleware(thread_limit=50, run_limit=50),
        ToolCallLimitMiddleware(
            tool_name="web_search", thread_limit=20, run_limit=5
        ),
        ToolRetryMiddleware(max_retries=3, backoff_factor=2.0, initial_delay=1.0),
    ]

    if extra_middleware:
        user_middleware = user_middleware + list(extra_middleware)

    agent = create_deep_agent(
        model=langchain_llm,
        tools=tools,
        system_prompt=system_prompt,
        middleware=user_middleware,
        subagents=subagents or [],
        backend=backend,
        context_schema=context_schema,
        checkpointer=checkpointer,
        store=store,
        memory=["/memories/AGENTS.md"],
        interrupt_on=interrupt_on,
    )
    return agent
```

**能力前缀解析逻辑** (`triage_prompt.py`):

```python
# 能力前缀 → Specialist 映射
CAPABILITY_PREFIX_MAP = {
    "生成ppt:": "ppt_specialist",
    "深度研究:": "research_specialist",
    "分析文档:": "file_analysis_specialist",
    "数据分析:": "data_analysis_specialist",
    "代码审查:": "code_review_specialist",
    "写代码:": "coding_specialist",
}

def parse_capability_prefix(message: str) -> tuple[str | None, str]:
    """解析能力前缀，返回 (specialist_name, 清洗后的消息)。"""
    for prefix, specialist in CAPABILITY_PREFIX_MAP.items():
        if message.startswith(prefix):
            clean_msg = message[len(prefix):].strip()
            return specialist, clean_msg
    return None, message
```

```markdown
# PPT 生成专家

你是专业的 PPT 自动生成专家，负责将用户需求转化为结构化的 PowerPoint 演示文稿。

## 工作流程

1. **需求分析**：理解用户主题、受众、风格偏好
2. **结构规划**：设计幻灯片结构（封面→目录→内容→结束）
3. **大纲生成**：为每页生成标题和关键要点
4. **内容填充**：搜索相关资料，填充每页内容（自动搜索配图）
5. **渲染输出**：调用 ppt_render 工具渲染为 .pptx 文件

## 约束
```

```python
from deepagents import SubAgent
from src.apis_agent.tool import TOOL_REGISTRY

def discover_specialist_agents(subagents_dir: str | None = None) -> list[SubAgent]:
    """扫描 subagents/ 目录，构建 SubAgent 列表。
    
    SubAgent 通过 SubAgentMiddleware 注入为 task 工具，
    LLM 通过 Function Calling 自行选择。
    """
    if subagents_dir is None:
        subagents_dir = Path(__file__).resolve().parent.parent / "subagents"

    agents: list[SubAgent] = []
    for child in sorted(Path(subagents_dir).iterdir()):
        if not child.is_dir() or child.name.startswith("_"):
            continue
        
        agent_md = child / "AGENT.md"
        if not agent_md.exists():
            continue

        spec = _parse_agent_md(agent_md)
        
        agent_def: SubAgent = {
            "name": spec["name"],
            "description": spec["description"],
            "system_prompt": spec["system_prompt"],
        }

        # 按 allowed_tools 从 TOOL_REGISTRY 获取工具对象
        allowed_names = spec.get("allowed_tools", [])
        if allowed_names:
            agent_def["tools"] = [
                TOOL_REGISTRY[name]
                for name in allowed_names
                if name in TOOL_REGISTRY
            ]
        
        agents.append(agent_def)
    
    return agents
```

**GatewayMiddleware 实现**（新文件 `src/apis_agent/gateway/middleware.py`）：

```python
class GatewayMiddleware(AgentMiddleware):
    """基于 ModelGateway 的智能模型路由中间件。
    
    替换原来的 GatewayLLM 包装器，在 Agent 管线层工作，
    可以感知 Agent 执行状态并推送降级/恢复 SSE 事件。
    """

    def __init__(self, gateway: ModelGateway, role: ModelRole):
        self.gateway = gateway
        self.role = role

    async def awrap_model_call(self, request, handler):
        model_chain = self.gateway.get_model_chain(self.role)
        if not model_chain:
            return await handler(request)

        last_error = None
        for i, (model_name, model_instance) in enumerate(model_chain):
            try:
                start = time.monotonic()
                result = await handler(request.override(model=model_instance))
                latency = (time.monotonic() - start) * 1000
                await self.gateway.record_success(model_name, latency)
                # 降级恢复通知
                if i == 0:
                    await self._emit_restored()
                return result
            except Exception as e:
                await self.gateway.record_failure(model_name, str(e))
                last_error = e
                if status_handler := current_handler.get():
                    await status_handler.emit_degraded()

        raise last_error or RuntimeError("所有模型不可用")
```
